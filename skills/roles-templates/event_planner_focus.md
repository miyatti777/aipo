## 🎯 分解原則（Decomposition Principles）

### 1. イベントライフサイクル

**企画 → 準備 → 集客 → 当日 → 振り返り**

- 明確な目標設定
- 綿密な準備
- 効果的な集客
- スムーズな運営
- 学びの蓄積

### 2. 体験設計

**参加者体験を中心に**

- Before（期待感）
- During（体験価値）
- After（継続的なつながり）

### 3. リスク管理

**想定外への備え**

- 複数のシナリオ準備
- バックアッププラン
- 緊急時対応フロー

---

## 📐 フェーズ構造（Phase Structure）

### Phase 1: Planning（企画）

**目的**: イベントの企画設計

**典型的なタスク分解**:
- 目的・目標設定
- ターゲット定義
- コンセプト策定
- 日程・会場選定
- 予算計画

### Phase 2: Preparation（準備）

**目的**: イベントの準備

**典型的なタスク分解**:
- 登壇者・出演者調整
- プログラム・タイムテーブル
- 会場・機材手配
- 配布物・制作物準備
- リハーサル

### Phase 3: Promotion（集客）

**目的**: 参加者の集客

**典型的なタスク分解**:
- 告知ページ作成
- SNS・メール告知
- メディア・プレスリリース
- 参加者管理
- 問合せ対応

### Phase 4: Execution（当日運営）

**目的**: イベント当日の運営

**典型的なタスク分解**:
- 会場設営
- 受付運営
- 進行管理
- トラブル対応
- 記録（写真/動画/アンケート）

### Phase 5: Follow-up（振り返り）

**目的**: 効果測定と次への活用

**典型的なタスク分解**:
- アンケート集計・分析
- KPI評価
- コンテンツ二次利用
- 参加者フォローアップ
- 振り返り・教訓整理

## 🔑 重要な考慮事項

### SubLayer判定基準

- **イベント種別**: カンファレンス/セミナー/懇親会
- **開催形式**: オンライン/オフライン/ハイブリッド
- **規模別**: 大規模/中規模/小規模

### Task判定基準

- **企画系**: コンセプト、プログラム、予算
- **準備系**: 会場、登壇者、制作物
- **運営系**: 当日運営、受付、進行
