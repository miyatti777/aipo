## 🎯 分解原則（Decomposition Principles）

### 1. CRISP-DMプロセス

**Business Understanding → Data → Modeling → Evaluation → Deployment**

- ビジネス課題の理解
- データの理解と準備
- モデリング
- 評価
- 展開

### 2. 仮説駆動型分析

**仮説 → 検証 → 示唆**

- ビジネス仮説を明確化
- データで検証
- アクショナブルな示唆

### 3. 反復的プロセス

**探索 → モデル → 改善**

- 探索的データ分析（EDA）
- モデル構築と評価
- 継続的な改善

---

## 📐 フェーズ構造（Phase Structure）

### Phase 1: Problem Definition（課題定義）

**目的**: ビジネス課題とデータ分析の接続

**典型的なタスク分解**:
- ビジネス課題のヒアリング
- 分析目的の明確化
- KPI・成功指標定義
- データ要件の整理
- 分析計画策定

### Phase 2: Data Preparation（データ準備）

**目的**: 分析用データの準備

**典型的なタスク分解**:
- データソース特定・収集
- データ品質評価
- データクレンジング
- 特徴量エンジニアリング
- 探索的データ分析（EDA）

### Phase 3: Modeling（モデリング）

**目的**: 予測・分類モデルの構築

**典型的なタスク分解**:
- アルゴリズム選定
- モデル学習
- ハイパーパラメータチューニング
- 交差検証
- モデル解釈

### Phase 4: Evaluation & Deployment（評価・展開）

**目的**: モデルの評価とビジネス適用

**典型的なタスク分解**:
- モデル性能評価
- ビジネスインパクト評価
- レポート・可視化
- モデルのデプロイ
- モニタリング・再学習計画

## 🔑 重要な考慮事項

### SubLayer判定基準

- **異なる分析テーマ**: 顧客分析/売上予測など
- **異なるモデル**: 分類/回帰/クラスタリング
- **段階的な分析**: 基礎分析→高度分析

### Task判定基準

- **分析系**: EDA、統計分析、仮説検証
- **モデリング系**: 特徴量設計、モデル構築、チューニング
- **報告系**: 可視化、レポート、プレゼン
