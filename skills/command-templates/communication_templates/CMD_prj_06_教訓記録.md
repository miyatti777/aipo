# 06 終結: 教訓記録（Lessons Learned）

## 最初に質問（実行前に回答してください）

- プロジェクト名（必須）
- プロジェクト期間・成果（必須）
- うまくいったこと（必須）
- 課題・問題点（必須）
- 改善提案・教訓（必須）
- 次回への提言（必須）

## 目的

Rules（`basic/06_pmbok_closing.mdc`）に準拠し、プロジェクト完了時の教訓記録（Lessons Learned）を作成します。プロジェクトの成功要因・失敗要因を分析し、組織の知識資産として蓄積します。

## 必要入力

- プロジェクト名: 対象プロジェクトの名称
- プロジェクト概要: 期間・目標・成果・参加者
- 成功要因: うまくいったこと・成功のポイント
- 課題・問題: 発生した問題・課題・制約
- 根本原因: 問題の根本的な原因分析
- 改善提案: 今後の改善案・対策
- 教訓: 次回プロジェクトへの提言・学び

## 実行手順（Rules Steps）

```yaml
- trigger: "教訓記録|レッスンラーニド|レッスンずらーんど"
  steps:
    - name: "start_info_collection"
      action: "call basic/pmbok_closing.mdc => lessons_learned_questions"
      message: "教訓記録（Lessons Learned）を作成します。いくつか質問を行いますので、回答お願いします。"
    - name: "wait_user_responses"
      action: "wait_for_all_answers"
    - name: "confirm_document_creation"
      action: "confirm"
      message: "教訓記録ドラフトを作成してよろしいですか？"
    - name: "create_draft"
      action: "create_markdown_file"
      path: "Flow/Private/{{today}}/draft_lessons_learned.md"
      template_reference: "basic/pmbok_closing.mdc => lessons_learned_template"
      message: "draft_lessons_learned.md を作成しました。プロジェクト完了後の振り返りにお使いください。"
```

## 教訓記録の構成要素

### 📋 プロジェクト概要

- *基本情報: プロジェクト名・期間・予算・チーム*
- *目標: 当初設定した目標・成功基準*
- *成果: 実際に達成した成果・deliverable*
- *KPI: 主要な成果指標・実績値*

### ✅ 成功要因

- *プロセス: 効果的だった手法・プロセス*
- *チーム: チームワーク・コミュニケーション*
- *技術: 有効だった技術・ツール*
- *管理: 効果的だった管理手法*

### ⚠️ 課題・問題点

- *発生した問題: 具体的な問題・課題*
- *影響: プロジェクトへの影響・損失*
- *対応: 実際に取った対応・対策*
- *結果: 対応の効果・結果*

### 🔍 根本原因分析

- *なぜなぜ分析: 問題の根本原因追求*
- *要因分類: 人・プロセス・技術・環境*
- *システム要因: 組織・制度・文化的要因*
- *予防可能性: 事前に防げたかの分析*

### 💡 改善提案・教訓

- *プロセス改善: 開発・管理プロセスの改善*
- *組織改善: 組織・体制・役割の改善*
- *技術改善: 技術・ツール・環境の改善*
- *人材育成: スキル・知識・経験の向上*

## 教訓記録の活用方法

### 📚 組織学習

- *ナレッジベース: 組織の知識資産として蓄積*
- *ベストプラクティス: 成功事例の標準化*
- *リスク予防: 類似問題の予防策*
- *スキル向上: チーム・個人のスキル向上*

### 🎯 次回プロジェクト

- *計画改善: より精度の高い計画策定*
- *リスク管理: 事前のリスク識別・対策*
- *チーム編成: 効果的なチーム構成*
- *ツール選定: 適切なツール・技術選択*

### 🔄 継続改善

- *プロセス標準化: 効果的な手法の標準化*
- *テンプレート: 再利用可能なテンプレート作成*
- *チェックリスト: 品質確保のためのチェックリスト*
- *ガイドライン: 実践的なガイドライン策定*

## 教訓記録のベストプラクティス

### ✅ 効果的な教訓記録

- *具体的: 抽象的でなく具体的な記述*
- *客観的: 感情的でなく客観的な分析*
- *行動指向: 次回への具体的なアクション*
- *バランス: 成功・失敗の両面を記録*

### ❌ 避けるべき教訓記録

- *個人攻撃: 個人を責める内容*
- *抽象的: 曖昧で行動につながらない*
- *一面的: 失敗のみ・成功のみに偏る*
- *表面的: 根本原因に踏み込まない*

## 教訓の分類・整理

### 📊 カテゴリ別分類

- *技術: 技術選択・実装・品質*
- *プロセス: 開発手法・管理手法*
- *人・組織: チーム・コミュニケーション・スキル*
- *外部: 顧客・パートナー・市場環境*

### 🎯 重要度・影響度

- *クリティカル: 必ず改善すべき重要事項*
- *重要: 改善により大きな効果が期待*
- *推奨: 改善により一定の効果が期待*
- *参考: 参考程度・状況により判断*

- 次回プロジェクトの計画時に教訓を参照
- 定期的な教訓レビュー・アップデートの実施
