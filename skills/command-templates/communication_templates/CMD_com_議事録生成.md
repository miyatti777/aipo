# CMD-03: 議事録生成コマンド

## 📌 コマンド概要

**目的**: 文字起こしから構造化された議事録を生成し、適切な場所に配置

**用途**: Part1-3で使用（AIミーティングノートの紹介、デモ）

**実行時間**: 1分以内

---

## 📥 Input

### 必須項目

- **文字起こしページ** (必須): Notion AIミーティングノートで生成された文字起こしページ

### オプション項目

- **関連プロジェクト** (オプション): Stock/programs/[プロジェクト名] のプロジェクト名
    - 指定された場合はそのプロジェクトにmeetings/とlogs/にも配置

### 入力例

```jsx
@CMD-03_議事録生成 を実行してください

文字起こしページ: [このページを@mention]
関連プロジェクト: To-Doアプリ企画
```

または：

```jsx
この文字起こしから議事録を生成してください
```

(現在開いている文字起こしページを自動認識)

---

## 📤 Output

### 作成されるページ構造

### **パターン1: 関連プロジェクトなしの場合**

```jsx
Flow/
└── 202512/
    └── 2025-12-09/
        └── 📝 議事録: [議題名] (YYYY-MM-DD)
            - ミーティング情報
            - 議論内容
            - 決定事項
            - アクションアイテム
            - 次回の予定
            - 元の文字起こし (トグル)
```

### **パターン2: 関連プロジェクトありの場合**

```jsx
Flow/
└── 202512/
    └── 2025-12-09/
        └── 📝 議事録: [議題名] (YYYY-MM-DD) ⬅️ メイン議事録

Stock/
└── programs/
    └── [プロジェクト名]/
        ├── meetings/
        │   └── 📝 議事録まとめ: [議題名] (YYYY-MM-DD) ⮅️ 簡潔版
        └── logs/
            └── 📜 文字起こしログ: [議題名] (YYYY-MM-DD) ⮅️ 全文
```

## 📋 処理ロジック

### ステップ1: 文字起こしの解析

1. 指定された文字起こしページを読み込み
2. 以下の情報を抽出：
    - 議題（タイトル）
    - 参加者
    - 議論内容
    - 決定事項
    - アクションアイテム
    - 次回予定

### ステップ2: Flowにメイン議事録を作成

1. 今日の日付フォルダを取得 (Flow/YYYYMM/YYYY-MM-DD)
2. 議事録ページを作成：
    - タイトル: 📝 議事録: [議題名] (YYYY-MM-DD)
    - アイコン: 📝
    - 親: Flow/YYYYMM/YYYY-MM-DD
3. 内容を以下の形式で生成：

```markdown
# 📝 議事録: [議題名]

## 📅 ミーティング情報

- **日時**: YYYY-MM-DD HH:MM
- **参加者**: [@参加者名]
- **議題**: [議題名]

---

## 💬 議論内容

[主な議論ポイントを箱条書きで]

- 論点1
- 論点2
- 論点3

---

## ✅ 決定事項

1. [決定1]
2. [決定2]
3. [決定3]

---

## 📌 アクションアイテム

- [ ] **[@担当者]** [アクション内容] - 期限: [YYYY-MM-DD]
- [ ] **[@担当者]** [アクション内容] - 期限: [YYYY-MM-DD]

---

## 📆 次回の予定

- **日時**: [次回の日時]
- **議題**: [次回の議題]

---

▶ **元の文字起こし**

[文字起こし全文をトグル内に格納]
```

### ステップ3: 関連プロジェクトがある場合の処理

**関連プロジェクトが指定されている場合:**

1. Stock/programs/[プロジェクト名]/meetings/ フォルダを確認
    - なければ作成
2. **議事録まとめ**ページを作成：
    - タイトル: 📝 議事録まとめ: [議題名] (YYYY-MM-DD)
    - アイコン: 📝
    - 親: Stock/programs/[プロジェクト名]/meetings/
    - 内容: 決定事項とアクションアイテムを中心に簡潔にまとめたもの
3. Stock/programs/[プロジェクト名]/logs/ フォルダを確認
    - なければ作成
4. **文字起こしログ**ページを作成：
    - タイトル: 📜 文字起こしログ: [議題名] (YYYY-MM-DD)
    - アイコン: 📜
    - 親: Stock/programs/[プロジェクト名]/logs/
    - 内容: 文字起こし全文

### ステップ4: 完了メッセージ

**関連プロジェクトなしの場合:**

```jsx
✅ 議事録が生成されました！

作成場所：
📝 Flow/202512/2025-12-09/議事録: [議題名] (2025-12-09)

内容：
- ミーティング情報
- 議論内容
- 決定事項
- アクションアイテム
- 次回予定
- 元の文字起こし（トグル内）
```

**関連プロジェクトありの場合:**

```jsx
✅ 議事録が生成されました！

作成場所：
📝 Flow/202512/2025-12-09/議事録: [議題名] (2025-12-09)
📝 Stock/programs/To-Doアプリ企画/meetings/議事録まとめ: [議題名] (2025-12-09)
📜 Stock/programs/To-Doアプリ企画/logs/文字起こしログ: [議題名] (2025-12-09)

📍 整理完了：
- Flowにメイン議事録
- Stock/meetings/に簡潔まとめ
- Stock/logs/に文字起こし全文
```

## ⚠️ 注意事項

### エラーハンドリング

1. **文字起こしページが指定されていない場合**
    - エラーメッセージ: 「文字起こしページを指定してください。」
2. **Flowフォルダが存在しない場合**
    - エラーメッセージ: 「Flowフォルダが見つかりません。先にインストールコマンドを実行してください。」
3. **関連プロジェクトがStock/programsに存在しない場合**
    - 警告メッセージ: 「プロジェクト「[名前]」がStock/programs/に見つかりませんでした。Flowのみに議事録を作成しました。」
4. **権限エラーの場合**
    - エラーメッセージ: 「ページ作成に失敗しました。権限を確認してください。」

### 実行時間

- 目標: 1分以内
- 関連プロジェクトありの場合でも1分以内に完了

### 使用シーン

- **ミーティング直後**: 文字起こしから議事録を生成
- **プロジェクトMTG**: 関連プロジェクトを指定してStockにも配置
- **定例MTG**: Flowに時系列で管理
