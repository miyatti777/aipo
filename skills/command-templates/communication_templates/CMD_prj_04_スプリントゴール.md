# 04 実行: スプリントゴール

## 最初に質問（実行前に回答してください）

- プロジェクト名（必須）
- スプリント番号（必須・例：Sprint 1）
- スプリント期間（必須・例：2週間）
- スプリントの主要目標（必須）
- 含まれるユーザーストーリー（必須）
- 完了の定義（必須）

## 目的

Rules（`basic/04_pmbok_executing.mdc`）に準拠し、スプリントの目標を明確に定義します。チーム全体で共有する具体的で測定可能なゴールを設定し、スプリントの成功基準を明確化します。

## 必要入力

- プロジェクト名: 対象プロジェクトの名称
- スプリント番号: スプリントの識別番号
- スプリント期間: 開始日・終了日・期間
- 主要目標: このスプリントで達成したいこと
- ユーザーストーリー: 含まれる機能・作業項目
- 完了の定義: 何をもって完了とするか

## 実行手順（Rules Steps）

```yaml
- trigger: "スプリントゴール|Sprint Goal"
  priority: high
  steps:
    - name: "start_info_collection"
      action: "call basic/pmbok_executing.mdc => sprint_goal_questions"
      message: "【スプリントゴール】作成に必要な情報を収集します。以下の質問に回答してください。"
    - name: "wait_user_responses"
      action: "wait_for_all_answers"
      message: "必要な回答が揃うまで先に進みません。"
    - name: "confirm_document_creation" 
      action: "confirm"
      message: "入力内容でスプリントゴールを作成してよろしいですか？"
    - name: "create_draft"
      action: "create_markdown_file"
      path: "{{patterns.flow_private_date}}/draft_sprint_goal_{{sprint_number}}.md"
      template_reference: "basic/pmbok_executing.mdc => sprint_goal_template"
      message: "スプリントゴールのドラフトを作成しました。内容を確認し、必要に応じて修正してください。"
    - name: notify
      action: notify
      message: |
        スプリントゴールのドラフトを Flow に生成しました。修正後、
        「確定反映して」と入力すると Stock/programs/{{program_id}}/projects/{{project_id}}/documents/4_executing/sprint_goals/ に保存されます。
```

## スプリントゴールの構成要素

### 🎯 SMART目標

- *Specific（具体的）: 何を達成するかが明確*
- *Measurable（測定可能）: 進捗・完了が測定できる*
- *Achievable（達成可能）: 現実的に達成可能*
- *Relevant（関連性）: プロジェクト目標に関連*
- *Time-bound（期限明確）: スプリント期間内で完了*

### 📋 含まれる要素

- *ビジネス価値: このスプリントで提供する価値*
- *ユーザーストーリー: 実装する機能・作業*
- *受入基準: 各ストーリーの完了条件*
- *リスク・制約: 想定されるリスク・制約事項*
- *成功指標: 成功を測る指標・KPI*

### 🔄 スプリント管理

- *デイリースタンドアップ: 日次の進捗確認*
- *バーンダウンチャート: 残作業の可視化*
- *インペディメント: 障害・課題の管理*
- *スコープ調整: 必要に応じた調整*

## スプリントゴールのベストプラクティス

### ✅ 良いスプリントゴール

- *価値重視: 技術的タスクより顧客価値を重視*
- *チーム共有: 全メンバーが理解・共有している*
- *柔軟性: 手段は柔軟、目標は固定*
- *検証可能: デモ・テストで検証できる*

### ❌ 避けるべきスプリントゴール

- *タスクリスト: 単なる作業項目の羅列*
- *曖昧: 「改善する」「最適化する」等の曖昧な表現*
- *非現実的: スプリント期間で達成不可能*
- *技術偏重: 技術的な実装のみに焦点*

## 活用方法

### 🎯 チーム運営

- *フォーカス: チームの集中力向上*
- *優先順位: 作業の優先順位付け*
- *意思決定: スプリント中の判断基準*
- *モチベーション: チームの一体感醸成*

### 📊 進捗管理

- *可視化: 進捗の見える化*
- *早期発見: 問題の早期発見*
- *調整: 必要に応じたスコープ調整*
- *振り返り: スプリント終了時の評価*
