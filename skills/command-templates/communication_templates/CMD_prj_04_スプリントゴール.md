# 04 実行: スプリントゴール

## 最初に質問（実行前に回答してください）

- プロジェクト名（必須）
- スプリント番号（必須・例：Sprint 1）
- スプリント期間（必須・例：2週間）
- スプリントの主要目標（必須）
- 含まれるユーザーストーリー（必須）
- 完了の定義（必須）

## 目的

PMBOK実行フェーズの手法に基づき、スプリントの目標を明確に定義します。チーム全体で共有する具体的で測定可能なゴールを設定し、スプリントの成功基準を明確化します。

## 必要入力

- プロジェクト名: 対象プロジェクトの名称
- スプリント番号: スプリントの識別番号
- スプリント期間: 開始日・終了日・期間
- 主要目標: このスプリントで達成したいこと
- ユーザーストーリー: 含まれる機能・作業項目
- 完了の定義: 何をもって完了とするか

## スプリントゴールの構成要素

### 🎯 SMART目標

| 要素 | 説明 |
|------|------|
| **Specific（具体的）** | 何を達成するかが明確 |
| **Measurable（測定可能）** | 進捗・完了が測定できる |
| **Achievable（達成可能）** | 現実的に達成可能 |
| **Relevant（関連性）** | プロジェクト目標に関連 |
| **Time-bound（期限明確）** | スプリント期間内で完了 |

### 📋 含まれる要素

- **ビジネス価値**: このスプリントで提供する価値
- **ユーザーストーリー**: 実装する機能・作業
- **受入基準**: 各ストーリーの完了条件
- **リスク・制約**: 想定されるリスク・制約事項
- **成功指標**: 成功を測る指標・KPI

## スプリントゴールのベストプラクティス

### ✅ 良いスプリントゴール

- **価値重視**: 技術的タスクより顧客価値を重視
- **チーム共有**: 全メンバーが理解・共有している
- **柔軟性**: 手段は柔軟、目標は固定
- **検証可能**: デモ・テストで検証できる

### ❌ 避けるべきスプリントゴール

- **タスクリスト**: 単なる作業項目の羅列
- **曖昧**: 「改善する」「最適化する」等の曖昧な表現
- **非現実的**: スプリント期間で達成不可能
- **技術偏重**: 技術的な実装のみに焦点
