# Phase 1: 企画要件の確認

## 🎯 実行内容

ユーザーと対話し、以下を明確化する：

### 1.1 記事の基本情報

**AI質問項目：**

> 以下の情報を教えてください：
> 

> 
> 

> 1. **記事タイトル（仮）**：どんなタイトルを想定していますか？
> 

> 2. **テーマ**：何について書きますか？
> 

> 3. **きっかけ**：なぜこの記事を書くのですか？
> 

> 4. **目標文字数**：どのくらいの分量を想定していますか？
> 

> 5. **公開期限**：いつまでに公開しますか？
> 

### 1.2 ターゲット読者の定義

**AI質問項目：**

> ターゲット読者について教えてください：
> 

> 
> 

> 1. **誰に読んでほしいか**：具体的な読者像は？
> 

> 2. **読者の課題**：読者が抱えている課題は？
> 

> 3. **読者の前提知識**：どのくらいの知識レベルを想定しますか？
> 

> 4. **読後に期待する行動**：読んだ後、読者にどうなってほしいですか？
> 

### 1.3 メッセージの明確化

**AI質問項目：**

> この記事で伝えたいことを教えてください：
> 

> 
> 

> 1. **コアメッセージ**：一文で言うと何を伝えたいですか？
> 

> 2. **切り口**：どんな視点・角度から伝えますか？
> 

> 3. **差別化ポイント**：他の記事と何が違いますか？
> 

> 4. **参考にするインサイト**：リサーチで得た気づきの中で活用するものは？
>

# Phase 4: セクション管理Database の作成

## 🎯 実行内容

記事の各セクションを管理する**実際に動くDatabase**を作成する。

### 4.1 Database設計

**必須プロパティ：**

| プロパティ名 | 型 | 説明 |
| --- | --- | --- |
| セクション名 | title | セクションのタイトル |
| 順序 | number | 表示順（1, 2, 3...） |
| 目的 | text | このセクションで伝えること |
| 想定文字数 | number | 目標文字数 |
| 実際の文字数 | number | 執筆後の実際の文字数 |
| ステータス | status | 未着手/執筆中/完了/レビュー中 |
| キーポイント | text | このセクションの要点 |
| 使用例 | text | 具体例・エピソード |
| メモ | text | 執筆時のメモ |
| 作成日 | created_time | 登録日 |
| 最終更新日 | last_edited_time | 最終更新日 |

**ステータスオプション：**

- 📝 未着手
- ✍️ 執筆中
- ✅ 完了
- 👀 レビュー中
- 🔄 修正中

### 4.2 ビュー設計

**作成するビュー：**

1. **📋 構成順（Table）**
    - すべてのセクションを表示
    - ソート：順序（昇順）
    - 記事全体の流れを確認
2. **📊 ステータス別（Board）**
    - カラム：ステータス
    - カンバン形式で進捗管理
3. **✍️ 執筆待ち（Table）**
    - フィルター：ステータス = 未着手
    - これから書くセクション
4. **📈 進捗（Table）**
    - すべてのセクションを表示
    - 実際の文字数 / 想定文字数を表示
    - 進捗を可視化

## 💡 カスタマイズ例

### 長文記事（5000字以上）の場合

**追加要素：**

- セクションを章レベル・節レベルで階層化
- 各章ごとに小見出しを追加
- 図表・画像の挿入位置を明記

### 連載記事の場合

**追加要素：**

- 連載全体の構成
- 各回のテーマ・内容
- 前回・次回との繋がり

### インタビュー記事の場合

**追加要素：**

- インタビュイープロフィール
- 質問リスト
- 引用箇所の候補
