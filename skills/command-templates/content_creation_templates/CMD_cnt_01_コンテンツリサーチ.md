# Phase 2: リサーチ対象Database の作成

## 🎯 実行内容

リサーチ対象を管理する**実際に動くDatabase**を作成する。

### 2.1 Database設計

**必須プロパティ：**

| プロパティ名 | 型 | 説明 |
| --- | --- | --- |
| タイトル | title | コンテンツのタイトル |
| URL | url | 元記事・資料のURL |
| カテゴリ | select | コンテンツの分類 |
| 著者 | text | 著者名 |
| タグ | multi_select | キーワードタグ |
| ステータス | status | 未読/読了/分析済み |
| 優先度 | select | 高/中/低 |
| メモ | text | 簡単なメモ |
| インサイト数 | number | 抽出したインサイト数 |
| 読了日 | date | 読み終えた日付 |
| 作成日 | created_time | 登録日 |

**ステータスオプション：**

- 📚 未読
- 👀 読了
- ✅ 分析済み
- ⭐ 重要

### 2.2 ビュー設計

**作成するビュー：**

1. **📋 全対象一覧（Table）**
    - すべてのリサーチ対象を表示
    - グループ化：ステータス
2. **📂 カテゴリ別（Board）**
    - カラム：カテゴリ
    - カテゴリごとに整理
3. **⭐ 重要資料（Table）**
    - フィルター：優先度 = 高 OR ステータス = 重要
    - 必ず読むべき資料
4. **📚 未読リスト（Table）**
    - フィルター：ステータス = 未読
    - 読むべき資料の一覧
5. **✅ 分析完了（Table）**
    - フィルター：ステータス = 分析済み
    - インサイト抽出が完了したもの

# Phase 3: インサイトDatabase の作成

## 🎯 実行内容

リサーチから得られたインサイトを管理する**実際に動くDatabase**を作成する。

### 3.1 Database設計

**必須プロパティ：**

| プロパティ名 | 型 | 説明 |
| --- | --- | --- |
| インサイト | title | 気づき・学びの内容 |
| カテゴリ | select | インサイトの分類 |
| 参照元 | relation | リサーチ対象DBへのリンク |
| 引用 | text | 原文の引用 |
| 自分の解釈 | text | 自分なりの解釈・考察 |
| 活用案 | text | どう活用するか |
| タグ | multi_select | キーワードタグ |
| 重要度 | select | 高/中/低 |
| ステータス | status | 新規/検討中/活用済み |
| 作成日 | created_time | 登録日 |

**ステータスオプション：**

- 💡 新規
- 🤔 検討中
- ✅ 活用済み
- ⭐ 最重要

### 3.2 ビュー設計

**作成するビュー：**

1. **📋 全インサイト一覧（Table）**
    - すべてのインサイトを表示
    - グループ化：カテゴリ
2. **⭐ 重要インサイト（Table）**
    - フィルター：重要度 = 高 OR ステータス = 最重要
    - 特に重要な気づき
3. **📂 カテゴリ別（Board）**
    - カラム：カテゴリ
    - カテゴリごとに整理
4. **🏷️ タグ別（Table）**
    - グループ化：タグ
    - タグで横断検索
5. **📚 参照元別（Table）**
    - グループ化：参照元
    - どの資料から何を学んだか

# Phase 4: リサーチ実施とインサイト抽出

## 🎯 実行内容

実際にリサーチを実施し、インサイトを抽出する。

### 4.1 リサーチ対象の登録

ユーザーが提供した情報をもとに、リサーチ対象Databaseに登録する。

**登録内容例：**

- 記事タイトル
- URL
- カテゴリ
- 著者
- 優先度

### 4.2 インサイト抽出（最低5件）

ユーザーと対話しながら、または提供された情報から、インサイトを抽出する。

**インサイト抽出フォーマット：**

```
インサイト: [気づき・学びの一文]
カテゴリ: [分類]
引用: [原文]
自分の解釈: [考察]
活用案: [どう使うか]
```

### 4.3 テーマ候補の抽出

インサイトをもとに、コンテンツのテーマ候補を抽出する。

**テーマ候補リスト例：**

1. **テーマA**
    - 切り口：[視点]
    - 対象読者：[誰向け]
    - メッセージ：[何を伝えるか]
2. **テーマB**
    - 切り口：[視点]
    - 対象読者：[誰向け]
    - メッセージ：[何を伝えるか]
