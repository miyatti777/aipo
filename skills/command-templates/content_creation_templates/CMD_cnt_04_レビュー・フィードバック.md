# Phase 3: フィードバックDatabase の作成

## 🎯 実行内容

フィードバックを管理する**実際に動くDatabase**を作成する。

### 3.1 Database設計

**必須プロパティ：**

| プロパティ名 | 型 | 説明 |
| --- | --- | --- |
| フィードバック内容 | title | コメントの内容 |
| セクション | select | 該当するセクション |
| カテゴリ | select | 内容/文章/構成/その他 |
| レビュアー | person | コメントした人 |
| 優先度 | select | 必須/推奨/任意 |
| ステータス | status | 未対応/対応中/対応完了/対応不要 |
| 対応内容 | text | どう対応したか |
| 元の文章 | text | 修正前の文章 |
| 修正後の文章 | text | 修正後の文章 |
| メモ | text | 対応時のメモ |
| 作成日 | created_time | 登録日 |

**ステータスオプション：**

- 📝 未対応
- 🔄 対応中
- ✅ 対応完了
- ❌ 対応不要

### 3.2 ビュー設計

**作成するビュー：**

1. **📋 全フィードバック一覧（Table）**
    - すべてのフィードバックを表示
    - グループ化：ステータス
2. **🔴 必須対応（Table）**
    - フィルター：優先度 = 必須
    - 必ず対応すべき項目
3. **👤 レビュアー別（Board）**
    - カラム：レビュアー
    - 誰からのフィードバックか
4. **📂 セクション別（Board）**
    - カラム：セクション
    - どのセクションのフィードバックか
5. **📊 カテゴリ別（Table）**
    - グループ化：カテゴリ
    - フィードバックの種類別

# Phase 4: フィードバック収集と整理

## 🎯 実行内容

レビュアーからフィードバックを受け取り、Databaseに整理する。

### 4.1 フィードバックの登録

**登録方法：**

1. **Notionコメントの場合**
    - 各コメントをフィードバックDBに転記
    - セクション、カテゴリ、優先度を設定
2. **別途ドキュメントの場合**
    - 内容を読み取り、項目ごとに分解
    - フィードバックDBに登録

### 4.2 フィードバックの分類

**分類軸：**

- **カテゴリ**：内容/文章/構成/その他
- **優先度**：必須/推奨/任意
- **セクション**：該当箇所

### 4.3 対応方針の決定

**各フィードバックについて：**

1. **必須対応**
    - すぐに修正対応
    - ステータス：対応中
2. **推奨対応**
    - 時間があれば対応
    - 優先順位を判断
3. **任意対応**
    - 参考として記録
    - 今回は対応しない可能性も

# Phase 5: 修正対応と最終確認

## 🎯 実行内容

フィードバックに基づき記事を修正し、最終確認する。

### 5.1 修正作業

**修正手順：**

1. **優先度順に対応**
    - 必須 → 推奨 → 任意
2. **修正内容を記録**
    - 元の文章
    - 修正後の文章
    - 対応内容
3. **ステータス更新**
    - 対応完了したら「対応完了」に変更

### 5.2 最終確認チェックリスト

**確認項目：**

- [ ]  すべての必須フィードバックに対応
- [ ]  推奨フィードバックの対応判断完了
- [ ]  修正箇所の整合性確認
- [ ]  全体を通読して違和感がないか
- [ ]  文字数が目標範囲内か
- [ ]  タイトルと内容が合っているか

## 💡 レビューのコツ

### 効果的なフィードバックの受け方

1. **感情的にならない**
    - フィードバックは記事を良くするため
    - 批判ではなく改善提案
2. **すべて受け入れない**
    - 自分の判断も大切
    - 意図と合わない提案は丁寧に断る
3. **質問する**
    - 不明なフィードバックは確認
    - レビュアーの意図を理解

### レビュアーとの良い関係

1. **明確な依頼**
    - 何を見てほしいか明確に
    - 期限と所要時間を提示
2. **感謝を伝える**
    - 時間を使ってくれたことへの感謝
    - 具体的にどう役立ったか
3. **フィードバック**
    - レビューについてのフィードバック
    - より良いレビュープロセスへ
