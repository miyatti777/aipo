# CMD_cnt_06_プレゼンスライド作成（Notionページ分割方式）

最終更新日時: 2025年12月25日 9:06

# 概要

Notionのページ分割方式でプレゼンスライド資料を作成します。各スライドを子ページとして作成し、左メニューから切り替える形式で、50名規模の会場でも視認性の高いスライドを実現します。

## 入力変数

```yaml
variables:
  presentation_time: "90分"  # 発表時間
  script_page_url: ""  # 台本ページURL
  slide_count_target: "30〜40枚"  # 目標スライド枚数
  audience_size: "50名"  # 参加者規模
  venue_info: ""  # 会場情報
```

---

## Notionページ分割方式の特徴

### メリット

- **Notion内で完結**: 外部ツール不要、全てNotion上で作成・表示
- **左メニューからの切り替え**: スムーズなページ遷移、操作性が高い
- **全画面表示**: 各ページがスライド1枚として機能
- **Notionブロック活用**: Callout、Database、画像、動画など全ブロックタイプ使用可能
- **デモとの統合**: コマンドやDBへの直リンクを配置可能
- **リアルタイム編集**: 直前まで修正可能
- **視認性**: 大きなフォントとCalloutで50名規模でも見やすい

### 構造イメージ

```
📊 プレゼンスライド（親ページ）
├─ 📄 00_表紙
├─ 📄 01_アジェンダ
├─ 📄 02_Part1イントロ
├─ 📄 03_デモ1
├─ 📄 04_デモ2
├─ ...
└─ 📄 99_まとめ
```

## 実行フロー

### Phase 1: スライド構成の設計（3分）

**AI実行内容**

- 台本を元にスライド分割案を作成
- 各スライドの役割を定義
- セクションごとに番号を振る（00-, 10-, 20-, ...）

**スライド枚数の目安**

| 発表時間 | 推奨枚数 | 1枚あたり |
| --- | --- | --- |
| 5分 | 7〜10枚 | 30〜40秒 |
| 10分 | 10〜15枚 | 40〜60秒 |
| 20分 | 20〜30枚 | 40〜60秒 |
| 90分 | 30〜40枚 | 2〜3分 |

**成果物**: スライド構成案リスト

**👤 Human Checkpoint 1**

- スライド分割は適切か？
- セクション分けは明確か？

---

### Phase 2: 親ページ作成（2分）

**AI実行内容**

- プレゼンスライドの親ページを作成
- 目次形式で子ページへのリンクを配置
- メタ情報（日時、会場、参加者数）を記載

**親ページテンプレート**

```markdown
# 📊 [タイトル]

> **日時**: YYYY-MM-DD HH:MM-HH:MM
> **会場**: [会場名]
> **参加者**: 約XX名

---

## 📑 スライド一覧

### オープニング
<page>00_表紙</page>
<page>01_アジェンダ</page>

### Part 1
<page>10_Part1タイトル</page>
<page>11_デモ1</page>
...

### クロージング
<page>90_まとめ</page>
<page>99_Q&A</page>
```

---

### Phase 3: 各スライドページ作成（メイン作業）

**AI実行内容**

- 各スライドを子ページとして作成
- 番号付きタイトル（例: "03_NotionAIとは"）
- 大きな見出し + Callout + 箇条書き
- スピーカーノート（ページ下部にトグルで配置）

**スライドページテンプレート**

```markdown
# [見出し]

<callout icon="💡" color="blue_bg">
	**キーメッセージ**
	
	重要なポイントを大きく表示
</callout>

## 主要ポイント

- **ポイント1**: 説明
- **ポイント2**: 説明
- **ポイント3**: 説明

---

▶### 🎤 スピーカーノート

	**トーク内容**:
	「ここで話す内容...」
	
	**所要時間**: X分
	**次のスライド**: [次のページ名]
```

**スライドデザイン原則**

<aside>
🎨

**Notion特有の工夫**

1. **Calloutを積極活用** - 視覚的に目立つ、色で情報整理
2. **見出しは大きく** - # や ## で階層化
3. **箇条書きは3〜5個まで** - 情報量をコントロール
4. **画像・動画を活用** - デモのスクリーンショット等
5. **空白を恐れない** - 1スライド1メッセージ
6. **スピーカーノートは必ずトグル** - 投影時は非表示
</aside>

## プレゼン当日の操作方法

### 準備

1. 親ページを開く
2. 左サイドバーを展開しておく
3. 00_表紙ページを全画面表示（Ctrl/Cmd + ）

### プレゼン中

- **ページ切り替え**: 左サイドバーから次のページをクリック
- **全画面表示**: Ctrl/Cmd +  で切り替え
- **スピーカーノート**: トグルを開いて確認（投影時は非表示）

### Tips

- デモ実演時はリンク先に直接ジャンプ
- 質問対応時は該当スライドにすぐ戻れる
- リアルタイム編集可能（メモ追加など）
