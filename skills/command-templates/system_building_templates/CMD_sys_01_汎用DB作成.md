# 04 実装: 汎用Database作成

<aside>
🚨

## 🔴 最重要原則：必ず実際のDatabaseを作成する

このテンプレートの唯一の目的は「**実際に動くDatabase**」を作成すること。

❌ **禁止**

- DBスキーマ定義だけ書いて終わり
- 設計書だけ作って終わり
- 「後で作る」と言って終わり

✅ **必須**

- 実際のDatabase作成
- 必要なプロパティ設定
- 用途に合ったビュー作成
- サンプルデータ投入
</aside>

---

## 🎯 このテンプレートでできること

**ProductBacklog以外**の、あらゆる種類のDatabaseを設計・作成できます。

### 対応する用途

- **案件管理DB**: 営業案件、プロジェクト、契約等
- **メンバー管理DB**: チームメンバー、組織、ロール等
- **タスク管理DB**: 作業タスク、TODO、チケット等
- **ドキュメント管理DB**: Wiki、ガイド、資料等
- **リソース管理DB**: ツール、アセット、リソース等
- **その他**: 用途に応じて自由に設計

---

## 📋 実行前の質問（必須回答）

以下を明確にしてから実行してください：

### 1. DB名

- 例: 「案件管理DB」「メンバーDB」「タスクトラッカー」

### 2. 用途・目的

- 何を管理するDBか？
- どんな課題を解決するか？
- 誰が使うか？

### 3. 必須プロパティ

最低限必要なプロパティをリストアップ：

- 例（案件管理DB）: 案件名、顧客名、ステータス、担当者、金額、受注予定日
- 例（メンバーDB）: 名前、ロール、チーム、メール、入社日

### 4. 必要なビュー

用途に応じて選択：

- **Table**: 一覧表示（デフォルト）
- **Board**: かんばん方式（ステータス管理）
- **Timeline**: ガントチャート（日程管理）
- **Calendar**: カレンダー（日付管理）
- **Gallery**: カード表示（ビジュアル重視）
- **List**: シンプルリスト

### 5. 初期データ

サンプルデータを何件入れるか？（推奨: 3〜5件）

## 🔄 実行手順

### Phase 1: DB設計の確認

**AIの動作:**

```jsx
📋 DB設計を確認します

【DB名】案件管理DB
【用途】営業案件の進捗管理
【対象ユーザー】営業チーム

【必須プロパティ】
1. 案件名（title）
2. 顧客名（text）
3. ステータス（status）
   - オプション: リード → 商談中 → 受注 → 失注
4. 担当者（person）
5. 金額（number）- ¥表示
6. 受注予定日（date）
7. 最終更新日（last_edited_time）

【必須ビュー】
1. Table: 全体一覧
2. Board: ステータス別かんばん
3. Timeline: 受注予定日順タイムライン

【初期データ】3件のサンプル案件

この設計でDatabase作成を開始しますか？
```

**人間の確認:** 「OK」または修正依頼

### Phase 5: 運用ガイド作成（付属）

**AIの動作:**

```jsx
📖 運用ガイドを作成中...

Databaseページに以下を追加：

## 使い方ガイド

### 新規案件の追加
1. 「New」ボタンをクリック
2. 案件名と顧客名を入力
3. ステータスを「リード」に設定
4. 担当者を割り当て

### ステータス更新
- Board表示でドラッグ&ドロップで移動
- または各案件ページでステータスを変更

### よくある質問
Q: 失注後の案件はどうする？
A: ステータスを「失注」に変更。削除はしない。

✅ 運用ガイド追加完了
```

## ⚙️ AIへの実行指示

<aside>
🤖

**AIへの指示（このテンプレートが@メンションされたとき）**

### Phase 1: 設計確認

1. ユーザーから提供された情報をもとにDB設計を提案
2. 必須プロパティを確認
3. 必要なビューを確認
4. ユーザーの承認を待つ

### Phase 2: Database作成

1. **実際にDatabaseを作成する**（create-databaseツール使用）
2. すべての必須プロパティを設定
3. すべての必要なビューを作成
4. 作成完了を報告

### Phase 3: データ投入

1. **実際にサンプルデータを投入**（create-pagesツール使用）
2. 3〜5件のサンプルページを作成
3. 各プロパティに適切な値を設定

### Phase 4: 動作確認

1. Databaseが実際に作成されているか確認
2. 各ビューで正しく表示されるか確認
3. サンプルデータが正しく表示されるか確認

### Phase 5: ガイド作成

1. Databaseページ内に運用ガイドを追加
2. 使い方とFAQを記載

**重要な原則:**

- **必ず実際のDatabaseを作成する**（設計書だけで終わらない）
- **すべてのフェーズを完了する**（途中で終わらない）
- **動作確認を必ず行う**（作って終わりにしない）
