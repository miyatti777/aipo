# Phase 2: Wikiトップページと構造の作成

## 🎯 実行内容

Wikiのトップページとカテゴリ構造を**実際に作成**する。

### 2.1 Wikiトップページの作成

**作成する内容：**

```markdown
# 📚 [Wiki名]

> [Wikiの目的やミッションを簡潔に説明]

---

## 📋 クイックリンク

### 🔥 よく使うページ

- [ページ1]
- [ページ2]
- [ページ3]

### 🔍 検索

- ドキュメント検索Database

---

## 📂 カテゴリ

### [カテゴリ1アイコン] カテゴリ1名

[カテゴリ1ページ]

[カテゴリ1の説明]

### [カテゴリ2アイコン] カテゴリ2名

[カテゴリ2ページ]

[カテゴリ2の説明]

...

---

## 📚 ドキュメント検索

[ドキュメント管理DB - inline]

---

## 📝 最近の更新

- [YYYY-MM-DD] [更新内容]
- [YYYY-MM-DD] [更新内容]

---

## ❓ 困ったときは

- 🔍 検索機能でキーワード検索
- ❓ FAQページを確認
- 👥 [担当者]に問い合わせ
```

### 2.2 カテゴリページの作成

各カテゴリごとにページを作成し、以下の構造を持たせる：

```markdown
- [ドキュメント1]
- [ドキュメント2]

### [サブカテゴリ2]

- [ドキュメント3]
- [ドキュメント4]
```

# Phase 3: ドキュメント管理Databaseの作成

## 🎯 実行内容

Wiki内のドキュメントを管理する**実際に動くDatabase**を作成する。

### 3.1 Database設計

**必須プロパティ：**

| プロパティ名 | 型 | 説明 |
| --- | --- | --- |
| ドキュメント名 | title | ドキュメントの名称 |
| カテゴリ | select | ドキュメントの分類 |
| タグ | multi_select | 検索用タグ |
| ステータス | status | 下書き/レビュー中/公開/アーカイブ |
| 作成者 | person | ドキュメント作成者 |
| レビュアー | person | レビュー担当者 |
| 優先度 | select | 高/中/低 |
| 対象読者 | multi_select | 誰向けのドキュメントか |
| 最終レビュー日 | date | 最後に見直した日付 |
| ページリンク | url | 実際のドキュメントページへのリンク |
| 作成日 | created_time | ドキュメント作成日 |
| 最終更新日 | last_edited_time | 最終更新日 |

**ステータスオプション：**

- ✏️ 下書き
- 👀 レビュー中
- ✅ 公開
- 🔄 更新必要
- 📦 アーカイブ

### 3.2 ビュー設計

**作成するビュー：**

1. **📋 全ドキュメント一覧（Table）**
    - すべてのドキュメントを表示
    - グループ化：カテゴリ
2. **✅ 公開中（Table）**
    - フィルター：ステータス = 公開
    - 最もよく使うビュー
3. **📂 カテゴリ別（Board）**
    - カラム：カテゴリ
    - カテゴリごとに整理
4. **🔄 更新必要（Table）**
    - フィルター：最終レビュー日 < 3ヶ月前 OR ステータス = 更新必要
    - 定期見直しが必要なドキュメント
5. **👀 レビュー待ち（Table）**
    - フィルター：ステータス = レビュー中
    - レビューが必要なドキュメント
6. **🏷️ タグ別（Table）**
    - グループ化：タグ
    - タグで横断検索

# Phase 5: Wiki運用ガイドの作成

## 🎯 実行内容

Wiki全体の運用ルールとガイドラインを作成する。

### 5.1 運用ガイド構成

```markdown
# Wiki運用ガイド

## 📋 このWikiについて

**目的**：[Wikiの目的]
**対象者**：[対象ユーザー]
**管理者**：[管理担当者]

---

## ✏️ ドキュメントの作成・編集ルール

### 新規ドキュメント作成手順

1. **カテゴリを選択**
   - 適切なカテゴリフォルダ配下に作成

2. **テンプレートを使用**
   - テンプレート集から適切なテンプレートを選択

3. **Database登録**
   - ドキュメント管理DBに登録
   - カテゴリ、タグ、ステータスを設定

4. **レビュー依頼**
   - ステータスを「レビュー中」に変更
   - レビュアーをアサイン

5. **公開**
   - レビュー完了後、ステータスを「公開」に変更

### 編集ルール

- **軽微な修正**：直接編集してOK（誤字脱字、リンク修正等）
- **大幅な変更**：レビューが必要
- **削除**：アーカイブに変更（削除は原則しない）

---

## 🔍 ドキュメントの検索方法

### 方法1: Database検索

- トップページのドキュメント管理DBで検索
- カテゴリ、タグ、キーワードで絞り込み

### 方法2: Notion検索

- `Cmd + K` (Mac) / `Ctrl + K` (Windows)
- Wiki名で絞り込んでから検索

### 方法3: カテゴリから探す

- トップページのカテゴリから階層的に探す

---

## 📊 ドキュメントのステータス

| ステータス | 意味 | 誰が見る |
|----------|------|----------|
| ✏️ 下書き | 作成中 | 作成者のみ |
| 👀 レビュー中 | レビュー待ち | レビュアー |
| ✅ 公開 | 公開済み | 全員 |
| 🔄 更新必要 | 情報が古い | 全員（注意） |
| 📦 アーカイブ | 過去の資料 | 参照のみ |

---

## 🔄 ドキュメントの定期見直し

### 見直し頻度

- **重要ドキュメント**：3ヶ月ごと
- **一般ドキュメント**：6ヶ月ごと
- **アーカイブ**：見直し不要

### 見直しプロセス

1. 「更新必要」ビューで対象を確認
2. 内容を確認・更新
3. 最終レビュー日を更新
4. 不要な場合はアーカイブに変更

---

## 💡 ドキュメント作成のベストプラクティス

### タイトル

- 明確で検索しやすい名前
- 「〇〇の使い方」「〇〇ガイド」など統一

### 構造

- 見出しを適切に使用
- 箇条書きで読みやすく
- 長すぎる場合は分割

### タグ付け

- 検索を意識したタグ設定
- 最低2つ、最大5つ程度

### 更新履歴

- 大きな変更は履歴を残す
- 変更理由も記載

---

## 👥 役割と責任

| 役割 | 責任 |
|------|------|
| Wiki管理者 | Wiki全体の管理、構造改善 |
| カテゴリオーナー | カテゴリ内のドキュメント品質管理 |
| 作成者 | ドキュメント作成、更新 |
| レビュアー | 内容レビュー、承認 |
| 利用者 | ドキュメント活用、フィードバック |

---

## ❓ よくある質問

### Q: どのテンプレートを使えばいい？
A: テンプレート集を確認。迷ったら管理者に相談。

### Q: 古いドキュメントを見つけたら？
A: ステータスを「更新必要」に変更し、カテゴリオーナーに連絡。

### Q: 新しいカテゴリが必要な場合は？
A: Wiki管理者に相談。全体構造を考慮して追加。
```
