## 📝 実行フロー（5 Phase）

### Phase 1: 入力要件確認・UI設計（10分）

**目的**: 誰が何を入力するのか、どんな項目が必要かを明確化

**実行内容**:

```yaml
確認事項:
  - 入力者: 誰が使うUIか？（内部メンバー/外部ユーザー/両方）
  - 入力内容: 何を入力してもらうか？
  - 必須項目: 絶対に必要な入力は？
  - 任意項目: あると便利な入力は？
  - バリデーション: 入力制約はあるか？（文字数制限/必須チェック等）
  - 送信先: どのDatabaseに保存するか？

UI設計:
  - 入力項目リスト（名前/型/必須/説明）
  - 項目の並び順（論理的な順序）
  - 説明文・ヘルプテキスト
  - 送信後の動作（確認メッセージ/リダイレクト）
```

**チェックポイント**:

- [ ]  入力者と入力内容が明確に定義されている
- [ ]  入力項目リストが完成している
- [ ]  送信先Databaseが決まっている

---

### Phase 2: 対象Database準備（15分）

**目的**: FormがデータをForm送信するDatabaseを準備（既存or新規作成）

**実行内容**:

```yaml
既存DBを使う場合:
  - Databaseを確認
  - 必要なプロパティが揃っているか確認
  - 不足プロパティがあれば追加

新規DBを作成する場合:
  - 05_Database実装テンプレートを使用
  - 入力項目に対応するプロパティを設計
  - Database作成・プロパティ設定

プロパティ対応表作成:
  | Form入力項目 | Database Property | 型 | 必須 |
  |------------|------------------|-----|------|
  | 記事URL | article_url | URL | ○ |
  | テーマ | theme | Text | ○ |
  | 対象人物 | target_person | Text | × |
```

**チェックポイント**:

- [ ]  送信先Databaseが準備できている
- [ ]  Form入力項目とDatabaseプロパティの対応が明確

---

### Phase 3: Form作成・公開設定（20分）

**目的**: NotionのForm機能を使って実際のフォームを作成

**実行内容**:

```yaml
Form作成手順:
  1. 対象DatabaseのViewメニューから「Form」を選択
  2. Form Viewを作成
  3. Form設定:
     - タイトル設定（例: 「記事アイディア入力フォーム」）
     - 説明文設定（フォームの目的・使い方）
     - 入力項目の選択・並べ替え
     - 各項目の説明文・プレースホルダー設定
     - 必須項目の指定
  
  4. 公開設定:
     - 「Share to web」で公開URLを取得
     - アクセス権限設定（誰でも/ワークスペースメンバーのみ）
     - 送信後メッセージのカスタマイズ

Form設定例:
```

タイトル: ノッカリ記事アイディア入力

説明: お気に入りの記事URLとテーマを入力してください。

AIが構成案を自動生成します。

入力項目:

1. 記事URL（必須）
    - プレースホルダー: [https://note.com/xxx/n/xxxx](https://note.com/xxx/n/xxxx)
2. テーマ（必須）
    - プレースホルダー: 例: AIプロダクト開発の極意
3. 対象人物（任意）
    - プレースホルダー: 例: 柳川慶太

送信後メッセージ:

「ありがとうございます！構成案を生成中です…」

```

```

**Notion公式ドキュメント参照**:

- Form: [https://www.notion.com/ja/product/forms](https://www.notion.com/ja/product/forms)

**チェックポイント**:

- [ ]  Formが実際に作成されている
- [ ]  公開URLが取得できている
- [ ]  入力項目が正しく設定されている
- [ ]  説明文・プレースホルダーが設定されている

---

### Phase 4: Button作成・処理連携（15分）

**目的**: ページ内にButtonを配置し、AI処理や次のステップをトリガー

**実行内容**:

```yaml
Button作成（必要に応じて）:
  
  使用シーン:
    - Form送信後の処理トリガー
    - AI処理の実行ボタン
    - ステータス更新ボタン
    - 次のステップへの誘導
  
  Button設定例:
    1. AI処理実行ボタン:
```

Button名: 「構成案を生成」

実行内容: AI処理コマンドをトリガー

配置場所: Form送信完了ページ

```

2. ステータス更新ボタン:
```

Button名: 「処理完了にする」

実行内容: Statusプロパティを「完了」に更新

配置場所: 各データページ

```

3. 次ステップ誘導ボタン:
```

Button名: 「執筆を開始」

実行内容: 執筆ページを作成・リンク

配置場所: 企画完了後のページ

```

AI連携:
  - AI Blockとの組み合わせ
  - Operation Commandの呼び出し
  - 自動通知・リマインダー設定
```

**Notion公式ドキュメント参照**:

- Button: [https://www.notion.com/ja/help/button-block](https://www.notion.com/ja/help/button-block)

**チェックポイント**:

- [ ]  必要なButtonが配置されている
- [ ]  Buttonが正しく動作する
- [ ]  AI処理との連携が機能している

---

### Phase 5: 動作確認・ドキュメント整備（10分）

**目的**: 実際にFormを使ってテスト送信し、運用ガイドを作成

**実行内容**:

```yaml
動作確認:
  テストケース:
    1. 標準ケース（全項目入力）
    2. 最小ケース（必須項目のみ）
    3. エッジケース（長文/特殊文字）
  
  確認項目:
    - [ ] Form送信が成功する
    - [ ] Databaseに正しくデータが保存される
    - [ ] 必須項目のバリデーションが機能する
    - [ ] 送信後メッセージが表示される
    - [ ] Buttonが正しく動作する

運用ガイド作成:
  内容:
    - Form公開URL
    - Formの使い方（スクリーンショット付き）
    - 入力項目の説明・注意点
    - 送信後の流れ（どこに保存される/誰が確認する）
    - よくある問題と対処法
    
  配置場所:
    - WikiページにFormを埋め込み
    - SlackチャンネルにURL共有
    - オンボーディングドキュメントに追加
```

**チェックポイント**:

- [ ]  最低3回のテスト送信が成功している
- [ ]  運用ガイドが完成している
- [ ]  FormのURLが関係者に共有されている
