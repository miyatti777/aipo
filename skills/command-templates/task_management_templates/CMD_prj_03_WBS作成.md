# 03 計画: WBS作成（作業分解構造）

## 最初に質問（実行前に回答してください）

- プロジェクト名（必須）
- プロジェクトの主要成果物（必須）
- 作業の分解レベル（必須・例：3階層、4階層）
- 担当者・期間の情報（任意）

## 目的

PMBOK計画フェーズの手法に基づき、プロジェクトの作業分解構造（WBS: Work Breakdown Structure）を作成します。プロジェクトの全作業を階層的に整理し、管理可能な単位に分解します。

## 必要入力

- プロジェクト名: 対象プロジェクトの名称
- 主要成果物: プロジェクトで作成する主要な成果物・機能
- 分解レベル: 作業をどこまで細分化するか（通常3-4階層）
- 担当者情報: 各作業の担当者（分かる範囲で）
- 期間情報: 各作業の想定期間（分かる範囲で）

## WBS構造例

### 📋 階層構造

```
1. プロジェクト名
  1.1 フェーズ1（例：要件定義）
    1.1.1 作業パッケージ1（例：現状分析）
      1.1.1.1 作業項目1（例：ヒアリング実施）
      1.1.1.2 作業項目2（例：分析レポート作成）
    1.1.2 作業パッケージ2（例：要件整理）
  1.2 フェーズ2（例：設計）
    1.2.1 作業パッケージ3（例：基本設計）
    1.2.2 作業パッケージ4（例：詳細設計）
```

### 🎯 作業パッケージの特徴

| 特徴 | 説明 |
|------|------|
| **測定可能** | 進捗が測定できる |
| **管理可能** | 1人の担当者で管理できる規模 |
| **期間明確** | 開始・終了が明確 |
| **成果物明確** | 何を作るかが明確 |

## WBSテンプレート

```markdown
# WBS（作業分解構造）

## 基本情報
- **プロジェクト名**: [プロジェクト名]
- **作成日**: [YYYY-MM-DD]
- **バージョン**: 1.0

## WBS構造

### 1. [プロジェクト名]

#### 1.1 [フェーズ1名]
| WBS ID | 作業項目 | 担当者 | 想定工数 | 成果物 |
|--------|---------|--------|---------|--------|
| 1.1.1 | [作業名] | [名前] | [X日] | [成果物] |
| 1.1.2 | [作業名] | [名前] | [X日] | [成果物] |

#### 1.2 [フェーズ2名]
| WBS ID | 作業項目 | 担当者 | 想定工数 | 成果物 |
|--------|---------|--------|---------|--------|
| 1.2.1 | [作業名] | [名前] | [X日] | [成果物] |
| 1.2.2 | [作業名] | [名前] | [X日] | [成果物] |

## 依存関係

| 先行作業 | 後続作業 | 関係タイプ |
|---------|---------|-----------|
| 1.1.1 | 1.1.2 | FS（終了-開始） |
| 1.1.2 | 1.2.1 | FS（終了-開始） |
```

## WBSの活用方法

### 📊 プロジェクト管理

- **進捗管理**: 各作業パッケージの完了状況
- **リソース配分**: 担当者・工数の割り当て
- **スケジュール作成**: 作業順序・依存関係の整理

### 💰 コスト管理

- **見積もり**: 各作業の工数・コスト見積もり
- **予算管理**: 作業単位での予算配分
- **実績管理**: 計画vs実績の比較

### 🎯 品質管理

- **成果物定義**: 各作業で作成する成果物の明確化
- **品質基準**: 成果物の品質基準設定
- **レビューポイント**: 品質確認のタイミング設定
