# CMD_prs_01_プレゼン資料作成

最終更新日時: 2025年12月25日 9:06

## 最初に質問（実行前に回答してください）

- プレゼンのテーマは何ですか？
- 対象者は誰ですか？（経営層、チーム、クライアント等）
- 所要時間は？
- 伝えたい主なメッセージは？

## 目的

プロジェクトやプロダクトに関するプレゼンテーション資料を作成します。対象者に合わせた構成・内容で、説得力のあるプレゼンを支援します。

## 必要入力

- **テーマ**: プレゼンの主題
- **対象者**: 誰に向けて話すか
- **時間**: 所要時間（5分、15分、30分等）
- **目的**: 何を伝えたいか、何を決めてもらいたいか
- **参考資料**: 既存のドキュメント、データ等

## 実行手順

### ステップ1: 構成の設計

- ストーリーラインを設計
- スライド枚数を決定（1分あたり1-2枚）
- 主要メッセージを明確化

### ステップ2: コンテンツ作成

- Flow でドラフトを作成
- セクションごとに内容を整理
- データ・図表を準備

### ステップ3: ブラッシュアップ

- ストーリーの流れを確認
- メッセージの明確さをチェック
- ビジュアルの改善

## プレゼン資料の基本構成

### 📊 オープニング（10%）

1. **タイトルスライド**
    - テーマ
    - プレゼンター
    - 日付
2. **アジェンダ**
    - 今日話すこと
    - 所要時間

### 🎯 本編（80%）

1. **背景・課題**
    - なぜこのテーマなのか
    - 何が問題なのか
2. **提案・解決策**
    - どう解決するのか
    - なぜこの方法なのか
3. **効果・インパクト**
    - 何が変わるのか
    - どんな価値があるのか
4. **実行計画**
    - いつまでに何をするか
    - 必要なリソースは

### ✅ クロージング（10%）

1. **まとめ**
    - 主要メッセージの再確認
    - アクションアイテム
2. **Q&A**
    - 質疑応答

## 対象者別のポイント

### 経営層向け

- **結論ファースト**: 最初に結論を提示
- **数字で語る**: ROI、KPI等の定量データ
- **簡潔に**: 詳細は補足資料へ
- **意思決定を促す**: 明確なアスク

### チーム向け

- **コンテキスト共有**: 背景を丁寧に説明
- **具体的な手順**: 実装方法の詳細
- **Q&A重視**: 疑問点を解消
- **協力を求める**: チームワークを促進

### クライアント向け

- **価値提案**: クライアントのメリット
- **事例紹介**: 類似案件の成功事例
- **信頼構築**: 専門性をアピール
- **次のステップ**: 具体的な提案

## Flow/Stock運用

### Flow での作業

- Flow/YYYYMM/presentations でドラフトを作成
- 複数バージョンを管理
- レビューコメントを反映

- 過去のプレゼンをナレッジとして蓄積

## プレゼンのTips

### 📝 スライド作成

- **1スライド1メッセージ**: 情報を詰め込みすぎない
- **視覚的に**: 図表を活用
- **読みやすく**: 大きなフォント、シンプルなデザイン

### 🎤 話し方

- **ストーリーで語る**: 論理だけでなく感情にも訴える
- **具体例を入れる**: 抽象論だけでは伝わらない
- **間を取る**: 重要なポイントの前後で間を取る

### ⏰ 時間管理

- **リハーサル**: 必ず事前に通しで練習
- **時間配分**: セクションごとの時間を決める
- **バッファ**: Q&A用の時間を確保

## 次に実行

- **リハーサル**: プレゼンの練習
- **フィードバック収集**: チームメンバーにレビュー依頼
- **資料の調整**: フィードバックを反映

## トラブルシュート

- 時間内に収まらない場合は内容を削減
- メッセージが伝わらない場合はストーリーを見直し
- スライドが多すぎる場合は統合・削除を検討
