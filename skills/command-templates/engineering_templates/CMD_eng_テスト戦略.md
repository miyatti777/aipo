## 📋 コマンド概要

```yaml
command: /eng/テスト戦略

category: engineering
related_role: qa_engineer_focus

description: |
  テスト戦略の策定と実行をサポートします。
```

## 📝 実行フロー

### Phase 1: テスト戦略策定

```markdown
## 📋 テスト戦略

### プロジェクト概要
- システム名: 
- 開発方式: [ ] ウォーターフォール [ ] アジャイル
- リリースサイクル: 

### 品質目標
| 指標 | 目標値 |
|------|-------|
| バグ流出率 | 件/リリース以下 |
| テストカバレッジ | %以上 |
| 自動テスト比率 | %以上 |

### テストピラミッド
```
        /\
       /  \      E2E
      /────\     (少)
     /      \
    /────────\   Integration
   /          \  (中)
  /────────────\ Unit
 /______________\(多)
```

### テストレベル
| レベル | 目的 | 担当 | 自動化 |
|--------|------|------|-------|
| Unit | 単体機能の検証 | 開発者 | ○ |
| Integration | 結合の検証 | 開発/QA | ○ |
| System | システム全体 | QA | △ |
| E2E | ユーザーシナリオ | QA | ○ |
| 受入 | 要件充足 | PO/顧客 | × |
```

### Phase 3: 自動化計画

```markdown
## 🤖 テスト自動化計画

### 自動化対象選定
| テスト種別 | 自動化 | ツール | 理由 |
|-----------|-------|-------|------|
| Unit | ○ | Jest | 必須 |
| API | ○ | Postman | 回帰用 |
| E2E | ○ | Playwright | 主要フロー |
| Visual | △ | Percy | 重要画面のみ |

### 自動化優先度
| 優先度 | 対象 | 理由 |
|--------|------|------|
| P1 | ログイン/認証 | 毎回使用 |

| P2 | 設定画面 | 変更頻度低 |
```
