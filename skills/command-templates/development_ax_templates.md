# Development AX Templates

開発者体験（Developer Experience）を向上させるコマンドテンプレート集

## 概要

「コード → 仕様 → テスト」の開発全工程を支援し、品質と生産性を同時に向上させるテンプレートです。
既存コードの解析から設計書作成、テスト計画、コードレビューまで、開発ライフサイクル全体をカバーします。

**特徴**:
- 🔍 **コード解析**: 既存コードベースを自動解析し、技術的負債を可視化
- 📝 **設計書自動生成**: 要件とコードから高品質なDesignDocを生成（HITL対応）
- 🧪 **テスト計画自動化**: リスクベースでテストケースを網羅的に生成
- ✅ **レビュー標準化**: プロジェクト固有のコードレビューチェックリスト作成

## 使用するRole

このテンプレート群は、以下の専門Roleを活用します：

### 🔍 role_code_analyst（コード解析スペシャリスト）
- 既存コードベースの構造解析
- 依存関係の可視化
- 技術的負債の評価
- コードメトリクスの測定

**使用テンプレート**: CMD_dv_10, CMD_dv_12

### 🏗️ role_software_architect（ソフトウェアアーキテクト）
- システム設計・DesignDoc作成
- アーキテクチャ図の生成
- API仕様・データモデルの設計
- ADR（設計判断記録）の作成

**使用テンプレート**: CMD_dv_10

### 🧪 role_qa_engineer（QAエンジニア）
- テスト計画の策定
- テストケースの生成
- テスト自動化戦略の立案
- CI/CD統合設計

**使用テンプレート**: CMD_dv_11, CMD_dv_12

## HITL（Human-in-the-Loop）について

全てのテンプレートで**Human-in-the-Loop**を採用しています。

### HITLのメリット

1. **品質保証**: 重要な判断に人間のレビューが入る
2. **カスタマイズ**: チーム固有の要件を反映できる
3. **学習機会**: AIの提案を確認することで知見が蓄積される
4. **信頼性**: 100%自動化ではなく、人間が最終判断を行う

### HITLポイント

| テンプレート | HITL-1 | HITL-2 |
|-------------|--------|--------|
| CMD_dv_10 | コンテキスト確認（5分） | 設計レビュー（10分） |
| CMD_dv_11 | テスト範囲確認（5分） | テストケースレビュー（10分） |
| CMD_dv_12 | チェック項目カスタマイズ（10分） | - |

## 活用シーン

### シーン1: 新機能追加時
```
1. CMD_dv_10 で設計書を作成
2. CMD_dv_11 でテスト計画を作成
3. 実装開始
4. CMD_dv_12 のチェックリストでレビュー
```

### シーン2: レガシーコードの改善
```
1. role_code_analyst でコード解析（内部処理）
2. CMD_dv_10 でリファクタリング設計書を作成
3. CMD_dv_11 で回帰テスト計画を作成
4. 段階的にリファクタリング実施
```

### シーン3: 品質基準の標準化
```
1. CMD_dv_12 でプロジェクト固有のチェックリスト作成
2. PRテンプレートに組み込み
3. CI/CDで自動チェック可能な項目を実装
4. チームでレビュー品質を向上
```

## よくある質問（FAQ）

### Q1: 既存のドキュメントがない場合でも使えますか？
**A**: はい。コードベースのみから設計書を逆生成できます。

### Q2: 他の言語・フレームワークに対応していますか？
**A**: テンプレートは汎用的に設計されており、主要な言語・フレームワークに対応可能です。

### Q3: HITLでの人間のレビュー時間はどのくらいですか？
**A**: 各テンプレートで合計15-20分程度を想定しています。

### Q4: 既存の設計書テンプレートと併用できますか？
**A**: はい。生成されたDesignDocをベースに、チーム独自のフォーマットに調整可能です。
