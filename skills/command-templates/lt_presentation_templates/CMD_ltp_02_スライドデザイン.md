# CMD_ltp_02_スライドデザイン

最終更新日時: 2025年12月25日 9:06

# 概要

LT・プレゼンテーションのスライド資料を作成します。「1スライド1〜2ポイント」の原則に基づき、視覚的にわかりやすい資料を設計します。

## 入力変数

```yaml
variables:
  presentation_time: "10分"  # 発表時間
  script_page_url: ""  # 台本ページURL（T001の成果物）
  slide_count_target: "10〜15枚"  # 目標スライド枚数
  design_tool: "Notion / Slides / Keynote"  # 使用ツール
```

---

## 実行フロー

### Phase 1: スライド構成の設計

**AI実行内容**

- 台本を元にスライド分割案を作成
- 各スライドの役割を定義
- 1スライド1〜2ポイントの原則を適用

**スライド枚数の目安**

| 発表時間 | 推奨枚数 | 1枚あたり |
| --- | --- | --- |
| 5分 | 7〜10枚 | 30〜40秒 |
| 10分 | 10〜15枚 | 40〜60秒 |
| 20分 | 20〜30枚 | 40〜60秒 |
| 40分 | 40〜60枚 | 40〜60秒 |

**成果物**: スライド構成案（Documentsフォルダ）

**👤 Human Checkpoint 1**

- スライド分割は適切か？
- 各スライドの役割は明確か？

---

### Phase 2: スライド下書きの作成

**AI実行内容**

- 各スライドのテキストを作成
- ビジュアル要素の提案（図・表・アイコン）
- レイアウト案の提示

**スライドデザイン原則**

<aside>
🎨

**1スライド1〜2ポイントまで**

- 情報を詰め込みすぎない
- 話す内容の「見出し」だけを書く
- 詳細は口頭で補う
</aside>

<aside>
📊

**イメージからキーワードへ**

- 文章ではなくキーワード
- 図・表・アイコンを活用
- 色は3色まで
</aside>

<aside>
✍️

**理解できる文章ではなく、次に進める文章**

- スライドだけで完結させない
- 「もっと聞きたい」と思わせる
- 過程は書かず、結論を示す
</aside>

**成果物**: スライド下書き（各スライドのテキスト案）

**👤 Human Checkpoint 2**

- テキストは簡潔か？
- ビジュアル要素は適切か？
- デザインの方向性は合っているか？

---

### Phase 3: スライド実装

**人間実行内容**

- 実際のスライドツールで作成
- ビジュアルデザインの調整
- フォント・色・レイアウトの最終決定

**AIサポート内容**

- 図表のデータ整理
- アイコン・画像の提案
- レイアウトの改善案

**成果物**: 完成スライド

**👤 Human Checkpoint 3**

- 全体の見た目は統一されているか？
- 決めゼリフのスライドは印象的か？

---

## 成果物

### 1. スライド構成案ページ

- スライド分割案
- 各スライドの役割定義

### 2. スライド下書きページ

- 各スライドのテキスト
- ビジュアル要素の提案

### 3. 完成スライド（外部ツール）

- 実際のプレゼンテーション資料

---

## 特殊スライドの設計

### 表紙スライド

- タイトル
- 発表者名
- イベント名・日付
- インパクトのある画像

### 決めゼリフスライド

- 大きなフォントで1行
- 装飾を最小限に
- 印象的な色使い

### 最終スライド

- 決めゼリフの再掲
- 連絡先（必要に応じて）
- 参考資料へのリンク
