# CMD_ltp_04_参考資料まとめ

最終更新日時: 2025年12月25日 9:06

# 概要

引用元・データソース・参考文献を整理し、信頼性を担保します。質疑応答時の補足資料としても活用します。

## 入力変数

```yaml
variables:
  citation_style: "URL / 文献形式 / 口頭のみ"  # 引用スタイル
  reference_page_create: true  # 参考資料ページを作成するか
  qanda_prep: true  # Q&A用の補足資料を準備するか
```

---

## 実行フロー

### Phase 1: 引用元の洗い出し

**AI実行内容**

- 台本・スライドで使用している情報源を特定
- データ・統計・事例の出典を整理
- 外部資料への参照を一覧化

**洗い出し対象**

- 統計データ・数字
- 研究論文・技術記事
- 事例・実装例
- 引用・参考にした資料

**成果物**: 引用元リスト（ドラフト）

**👤 Human Checkpoint 1**

- 引用元は正確か？
- 追加すべき資料はあるか？

---

### Phase 2: 参考資料の整理

**AI実行内容**

- 引用元を整形（URL・文献形式）
- カテゴリ分類（統計・論文・記事・事例等）
- アクセス可能性の確認（リンク切れチェック）

**整理フォーマット**

| カテゴリ | タイトル | URL | 引用箇所 |
| --- | --- | --- | --- |
| 統計 | 〇〇レポート2025 | https://... | スライド3 |
| 論文 | △△に関する研究 | https://... | 台本P2 |

**成果物**: 整理済み参考資料リスト

**👤 Human Checkpoint 2**

- 分類は適切か？
- リンクは全て有効か？

---

### Phase 3: Q&A補足資料の作成

**AI実行内容**（オプション）

- 想定質問のリストアップ
- 各質問への回答案と参考資料の紐付け
- 補足説明用の資料作成

**想定質問の例**

- 技術的な詳細
- パフォーマンス・コスト
- 実装の難易度
- 他の手法との比較

**成果物**: Q&A補足資料（オプション）

**👤 Human Checkpoint 3**

- 想定質問は適切か？
- 回答案は十分か？

---

## 成果物

### 1. 参考資料まとめページ

- 引用元リスト（カテゴリ別）
- URL・文献情報
- 引用箇所の対応表

### 2. Q&A補足資料（オプション）

- 想定質問リスト
- 回答案と参考資料

---

## 引用のベストプラクティス

<aside>
✅

**信頼性を高める引用**

- 公式データ・統計を優先
- 信頼できる情報源（論文・公式サイト）
- 最新の情報（発行年を確認）
</aside>

<aside>
📌

**引用の表示方法**

- スライドの隅に小さく表示
- 口頭で「〇〇の調査によると」と言及
- 最終スライドに参考資料一覧を掲載
</aside>

<aside>
🔍

**Q&A対策**

- 詳細データは参考資料で補完
- 「後ほど資料をお送りします」の準備
- スライドに載せきれない情報の整理
</aside>
