# CMD_ltp_05_リハーサル・改善

最終更新日時: 2025年12月25日 9:06

# 概要

実際に発表練習を行い、時間調整と改善を実施します。10分厳守（または指定時間）のタイミング確認が最重要です。

## 入力変数

```yaml
variables:
  presentation_time: "10分"  # 目標時間
  rehearsal_count: 2  # リハーサル回数
  recording: true  # 録画するか
  feedback_source: "自己評価 / 同僚 / AI"  # フィードバック元
```

---

## 実行フロー

### Phase 1: 第1回リハーサル

**人間実行内容**

- 実際に声に出して発表
- 時間を計測
- 引っかかる箇所・改善点をメモ

**計測ポイント**

- 全体の時間
- セクションごとの時間
- デモの時間
- 質疑応答の想定時間（ある場合）

**チェックリスト**

- [ ]  スムーズに話せるか
- [ ]  スライドと話す内容のタイミングは合っているか
- [ ]  デモは時間内に収まるか
- [ ]  決めゼリフは印象的に話せているか
- [ ]  目標時間に収まっているか（±1分以内）

**成果物**: 第1回リハーサル記録

**👤 Human Checkpoint 1**

- 時間は目標に収まっているか？
- 改善が必要な箇所はどこか？

---

### Phase 2: 改善実施

**AI実行内容**

- 時間オーバーの場合の削減案提示
- 話しにくい箇所の言い回し改善
- スライドの調整提案

**改善パターン**

| 問題 | 改善策 |
| --- | --- |
| 時間オーバー（+2分以上） | セクション削除、詳細説明のカット |
| 時間オーバー（+1分以内） | 話すスピード調整、冗長表現のカット |
| 時間アンダー（-2分以上） | 具体例追加、デモ時間延長 |
| 時間アンダー（-1分以内） | 間を取る、ゆっくり話す |
| 話しにくい箇所 | 言い回し変更、スライド文言調整 |

**成果物**: 改善計画・調整版台本

**👤 Human Checkpoint 2**

- 改善案は適切か？
- 台本・スライドの調整は完了したか？

---

### Phase 3: 第2回リハーサル

**人間実行内容**

- 改善を反映して再度発表
- 時間を再計測
- 最終調整ポイントを特定

**最終確認ポイント**

- [ ]  目標時間±30秒以内に収まっているか
- [ ]  スムーズに話せるか
- [ ]  デモは安定して動作するか
- [ ]  決めゼリフは自信を持って言えるか
- [ ]  全体の流れに違和感がないか

**成果物**: 第2回リハーサル記録

**👤 Human Checkpoint 3**

- 発表準備は完了したか？
- 追加のリハーサルは必要か？

---

## リハーサルのベストプラクティス

<aside>
⏱️

**時間厳守が最優先**

- LTは時間制限が厳しい
- オーバーすると途中で切られる
- 余裕を持って±30秒以内に収める
</aside>

<aside>
🎤

**実際に声に出して練習**

- 黙読と発声では時間が違う
- 話すと引っかかる箇所が分かる
- 決めゼリフは何度も練習
</aside>

<aside>
📹

**録画して自己レビュー**

- 話し方のクセが分かる
- スライド切り替えのタイミング確認
- 改善点の可視化
</aside>

<aside>
👥

**フィードバックをもらう**

- 同僚に聞いてもらう
- 分かりにくい箇所を特定
- 客観的な視点で改善
</aside>
