# CMD-04: チーム企画統合コマンド

**コマンド名**: `/team_integration`

**優先度**: P0（最重要）

**用途**: Part2-4（チーム企画統合、3分、ハンズオン）

**実行時間**: 3分以内

---

## 📌 コマンド概要

Part1で各自が作成した個人企画（ペルソナ・課題・ソリューション）を統合し、PMBOK構造でチームフォルダを作成するコマンドです。

**目的**:

- 3-5個の個人企画を論理的に統合
- チーム全体の統合プロジェクト憲章を生成
- マイベストの記事制作に特化した構造で出力

---

## 📥 Input

### 必須項目

1. **個人企画ページURL**（3-5個）
    - Part1のAIPM Discovery成果物
    - ペルソナ、課題定義、ソリューションマップが含まれる
2. **チーム名**
    - 例：チーム1、チーム2、マーケティングチーム

### 入力例

```jsx
@CMD-04_チーム企画統合コマンド を実行してください

【入力情報】
チーム名: チーム1

個人企画:
1. @田中さんのTo-Do企画
2. @佐藤さんのTo-Do企画
3. @鈴木さんのTo-Do企画
```

## 🔧 実行方法

### Step 1: 個人企画ページを準備

Part1で作成した以下のページをチームで共有：

- ペルソナ作成ページ
- 課題定義ページ
- ソリューションマップページ

### Step 2: コマンド実行

AIアシスタントに以下のように依頼：

```
CMD-04_チーム企画統合コマンド を実行してください

チーム名: [あなたのチーム名]

個人企画:
1. [メンバー1のファイルパス]
2. [メンバー2のファイルパス]
3. [メンバー3のファイルパス]
```

### Step 3: 結果確認

生成されたフォルダ構造を確認し、内容を精査します。

---

## 📋 処理ロジック

### Phase 1: 個人企画の読み込み（30秒）

1. 指定された個人企画ページを全て読み込む
2. 各ページからペルソナ・課題・ソリューション情報を抽出
3. データを構造化して保持

### Phase 2: 統合分析（90秒）

### 2.1 ペルソナ統合

- 各個人企画のペルソナを比較分析
- 共通点と差異を抽出
- 統合ペルソナ像を作成（年齢層、職業、ライフスタイル、課題認識）
- マイベストの読者層との整合性を確認

### 2.2 課題統合

- 各企画で定義された課題を分類
- 優先度の高い課題を抽出
- 課題間の関連性を整理
- 統合課題マップを作成

### 2.3 ソリューション統合

- 各企画のソリューションを機能ごとに分類
- 重複機能の統廃合
- 優先順位の設定（MUSt/SHOULD/COULD）
- 統合ソリューションマップを作成

### Phase 3: PMBOK構造生成（60秒）

1. チームフォルダを作成
2. [README.md](http://README.md)に統合企画書を生成
3. documents配下に3つのサブフォルダを作成
4. 各フォルダに詳細分析ドキュメントを配置

## 💡 マイベスト記事制作への応用例

### 例：記事企画統合シナリオ

**個人企画1（田中）**: 「30代ワーママ向けの時短レシピ記事」

- ペルソナ: 30代、共働き、2児の母
- 課題: 平日の夕食準備に時間がかけられない
- ソリューション: 15分以内で作れるレシピ、作り置き活用

**個人企画2（佐藤）**: 「料理初心者向けの簡単レシピ記事」

- ペルソナ: 20代、一人暮らし、料理経験なし
- 課題: 何を作れば良いかわからない、失敗が怖い
- ソリューション: 手順が少ない、失敗しにくいレシピ

**統合結果**:

- **統合ペルソナ**: 「忙しくて料理に時間をかけられない人」
- **統合課題**: 「限られた時間で美味しい食事を作りたい」
- **統合ソリューション**: 「時短×簡単×失敗しない」をコンセプトにした記事シリーズ

## ⚠️ 注意事項

### 技術的制約

- 一度に読み込めるページは最大5個まで
- 各ページのコンテンツサイズが大きすぎる場合、タイムアウトの可能性あり
- @mentionは正確なページURLを使用すること

### 運用上の注意

- 個人企画の質が統合結果の質に直結します
- Part1で十分な情報を記載してもらうことが重要
- チーム内で事前に企画の方向性を軽く議論しておくと効果的
