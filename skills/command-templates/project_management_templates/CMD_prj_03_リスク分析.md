# 03 計画: リスク分析

## 最初に質問（実行前に回答してください）

- プロジェクト名（必須）
- 想定されるリスク要因（必須）
- リスクの影響度・発生確率（必須）
- 現在の対策・予防策（任意）

## 目的

PMBOK計画フェーズの手法に基づき、プロジェクトのリスク計画を策定します。リスクを特定・分析し、対応策を計画してリスク管理体制を構築します。

## 必要入力

- プロジェクト名: 対象プロジェクトの名称
- リスク要因: 想定されるリスク・問題・課題
- 影響度: リスクが発生した場合の影響の大きさ
- 発生確率: リスクが発生する可能性
- 対応策: 予防策・軽減策・回避策・受容策

## リスク分析の構成要素

### 🎯 リスクの分類

| カテゴリ | 説明 | 例 |
|---------|------|-----|
| **技術リスク** | 技術的な課題・制約 | 新技術の習熟不足、パフォーマンス問題 |
| **スケジュールリスク** | 遅延・工期に関するリスク | 要件変更、見積もり精度 |
| **リソースリスク** | 人員・予算・設備に関するリスク | キーマン離脱、予算削減 |
| **外部リスク** | 市場・競合・規制等の外部要因 | 法規制変更、競合参入 |
| **組織リスク** | 組織変更・意思決定に関するリスク | スポンサー変更、優先度変更 |

### 📊 リスク評価マトリックス

```
        発生確率低    発生確率高
影響度高  [中リスク]   [高リスク]
影響度低  [低リスク]   [中リスク]
```

### 🛡 リスク対応戦略

| 戦略 | 説明 | 適用ケース |
|------|------|-----------|
| **回避（Avoid）** | リスクの原因を除去・回避 | 高影響・高確率のリスク |
| **軽減（Mitigate）** | リスクの影響・確率を軽減 | 対策可能なリスク |
| **転嫁（Transfer）** | リスクを第三者に転嫁 | 保険・外注で対応可能 |
| **受容（Accept）** | リスクを受け入れて監視 | 低影響または対策不可 |

## 活用方法

### 📊 プロジェクト監視

- 定期的なリスク状況の確認
- 新しいリスクの早期発見
- リスク対応策の効果測定

### 🎯 意思決定支援

- Go/No-Go判断の材料
- 追加投資の必要性判断
- スケジュール・スコープ調整の根拠

### 🔄 継続改善

- リスク対応の効果検証
- 次回プロジェクトへの教訓抽出
- 組織のリスク管理能力向上

## トラブルシュート

- リスクが多すぎる場合は、影響度・確率の高いものから優先的に対応
- 対応策が思い浮かばない場合は、類似プロジェクトの事例を参考
- 定量的な評価が困難な場合は、定性的な評価から開始
