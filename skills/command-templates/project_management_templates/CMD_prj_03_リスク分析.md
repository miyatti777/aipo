# 03 計画: リスク分析

## 最初に質問（実行前に回答してください）

- プロジェクト名（必須）
- 想定されるリスク要因（必須）
- リスクの影響度・発生確率（必須）
- 現在の対策・予防策（任意）

## 目的

Rules（`basic/03_pmbok_planning.mdc`）に準拠し、プロジェクトのリスク計画を策定します。リスクを特定・分析し、対応策を計画してリスク管理体制を構築します。

## 必要入力

- プロジェクト名: 対象プロジェクトの名称
- リスク要因: 想定されるリスク・問題・課題
- 影響度: リスクが発生した場合の影響の大きさ
- 発生確率: リスクが発生する可能性
- 対応策: 予防策・軽減策・回避策・受容策

## 実行手順（Rules Steps）

```yaml
- trigger: "リスク分析|リスク計画"
  steps:
    - name: "start_info_collection"
      action: "call basic/pmbok_planning.mdc => risk_questions"
      message: "【リスク分析】のための初期情報を収集します。回答をお願いします。"
    - name: "wait_user_responses"
      action: "wait_for_all_answers"
    - name: "confirm_document_creation"
      action: "confirm"
      message: "リスク計画をドラフト化してよろしいですか？"
    - name: "create_draft"
      action: "create_markdown_file"
      path: "{{patterns.draft_risk_plan}}"
      template_reference: "basic/pmbok_planning.mdc => risk_plan_template"
      message: "{{patterns.draft_risk_plan}} を作成しました。ご確認ください。"
```

## リスク分析の構成要素

### 🎯 リスクの分類

- *技術リスク: 技術的な課題・制約*
- *スケジュールリスク: 遅延・工期に関するリスク*
- *リソースリスク: 人員・予算・設備に関するリスク*
- *外部リスク: 市場・競合・規制等の外部要因*
- *組織リスク: 組織変更・意思決定に関するリスク*

### 📊 リスク評価マトリックス

```
        発生確率低    発生確率高
影響度高  [中リスク]   [高リスク]
影響度低  [低リスク]   [中リスク]
```

### 🛡 リスク対応戦略

- *回避（Avoid）: リスクの原因を除去・回避*
- *軽減（Mitigate）: リスクの影響・確率を軽減*
- *転嫁（Transfer）: リスクを第三者に転嫁*
- *受容（Accept）: リスクを受け入れて監視*

## リスク管理プロセス

### 🔍 リスク特定

- *ブレインストーミング: チームでのリスク洗い出し*
- *チェックリスト: 過去プロジェクトの教訓活用*
- *専門家判断: 専門家による評価*
- *SWOT分析: 強み・弱み・機会・脅威の分析*

### 📊 リスク分析

- *定性的分析: 影響度・確率の相対評価*
- *定量的分析: 数値による影響度算出*
- *感度分析: 各リスクの影響度比較*
- *シナリオ分析: 複数リスクの複合影響*

### 📋 リスク対応計画

- *予防策: リスク発生を防ぐ対策*
- *軽減策: 影響を小さくする対策*
- *緊急時対応: リスク発生時の対応手順*
- *責任者: 各リスクの監視・対応責任者*

## トラブルシュート

- リスクが多すぎる場合は、影響度・確率の高いものから優先的に対応
- 対応策が思い浮かばない場合は、類似プロジェクトの事例を参考
- 定量的な評価が困難な場合は、定性的な評価から開始
